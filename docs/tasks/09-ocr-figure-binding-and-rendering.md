# OCR 图块归属与宽容渲染规划

这份文档规划两条改进线：

1. OCR 图块归属以切题结果为主，避免跨页题图漂移到下一题。
2. 前端识别结果预览允许宽容渲染，入库和导出阶段继续保留严格校验。

当前先做规划，不直接修改实现。

## 背景问题

### 图块归属错位

以“衡阳一中2026届高三第一次月考”为例：

- 切题阶段 `CUT_0017` 已识别到第 17 题的题干图和解析图。
- 切题阶段 `CUT_0018` 没有题图。
- GLM OCR 原始版面中，第 15 页的顺序是：
  - 上方 image block：实际属于第 17 题解析。
  - 下方 text block：第 18 题题干开始。
- 当前 `_glm_figures()` 只按题目的 `page_span` 收集图片。第 18 题 page span 覆盖第 15 页，于是第 17 题解析图被错误归到第 18 题。

结论：只按页码范围归属图片不够，跨页题和页内题号边界会把图片推给错误题目。

### 公式渲染失败

以第 1 题为例，OCR/清洗后的解析中出现：

```tex
$ B=\left\{x\mid x\geq 2 $或 $ x<1\right\} $
```

这会把同一个集合公式切成两个数学片段：

- 前半段有 `\left\{`，没有配套的 `\right\}`。
- 后半段有 `\right\}`，没有配套的 `\left`。
- 中间的“或”脱离数学环境。

KaTeX 和后续 LaTeX 编译都无法稳定处理这种不平衡公式。

## 目标

### 图块归属目标

图块应优先服从切题复核阶段确认过的题号和题目区域。OCR 服务返回的图片只作为补充来源，不应覆盖或扩大切题边界。

### 渲染目标

前端查看识别结果时，不应该因为局部公式错误导致整段阅读体验崩坏；但入库和导出仍要保留严格的公式质量约束，避免坏 LaTeX 进入正式题库或导出 PDF。

## 非目标

- 本阶段不重写切题算法。
- 不把所有 OCR 输出强行清洗成完美 LaTeX。
- 不取消入库和导出的格式校验。
- 不把 GLM OCR 的图片结果完全丢弃，只降低它的归属优先级。

## 方案一：图块归属以切题边界为准

### 数据来源优先级

图块来源按以下顺序处理：

1. 切题复核记录中的 `figures_json`。
2. GLM OCR 返回的 image block，且必须能匹配到切题边界。
3. 用户在复核页手动新增或修正的图块。

第一优先级是切题复核记录。原因是切题阶段已经知道 `CUT_0017`、`CUT_0018`、题号、跨页段落和局部 bbox，这些信息比全文 OCR 的 page-level block 更接近用户确认过的题目边界。

### 归属规则

对每个 OCR image block，按以下规则判断归属：

1. 如果 image block 落在某个 review item 的 `segments_json` 范围内，则归属该 review item。
2. 如果多个 review item 都覆盖同一页，优先选择 bbox overlap 最大的 item。
3. 如果 image block 位于某题 segment 结束线之前、下一题题号起始线之前，则归属上一题。
4. 如果 image block 只在 page span 内，但不落入任何 segment 或题号边界，则不自动绑定，标记为 `unassigned_ocr_figure`。
5. 不允许仅凭 `page_span` 把图片归给题目。

### 跨页题处理

跨页题必须用 `segments_json` 判断。比如第 17 题覆盖第 13-15 页，第 18 题覆盖第 15-16 页时，第 15 页必须进一步比较：

- 第 17 题在第 15 页的 segment 范围。
- 第 18 题在第 15 页的起始位置。
- OCR image block 的 bbox 位置。

这样第 15 页上方属于第 17 题解析的图，不会因为第 18 题 page span 覆盖第 15 页而漂移过去。

### 导入策略

导入 OCR 草稿时：

1. 先读取 review item 的 `figures_json`。
2. 再筛选 GLM image blocks。
3. 只合并通过边界匹配的 GLM image blocks。
4. 未匹配图片写入诊断信息，不进入题目 `figures_json`。

建议诊断字段：

```json
{
  "figure_binding": {
    "source": "glm",
    "matched": 1,
    "unassigned": 2,
    "warnings": ["ocr_image_outside_review_segments"]
  }
}
```

## 方案二：前端宽容预览，入库/导出严格校验

### 分层原则

识别结果应分成三种形态：

1. 原始模型文本：尽量忠实保留 OCR 输出。
2. 前端预览文本：允许宽容渲染，局部失败不阻塞阅读。
3. 正式入库/导出文本：必须通过严格公式和 LaTeX 校验。

### 前端预览层

前端预览采用宽容策略：

- 能被 KaTeX 渲染的公式正常渲染。
- 不能渲染的公式降级为普通文本。
- 失败公式附近显示轻量状态，不让整段变红或破版。
- 保留模型原文，方便用户判断 OCR 内容是否可用。

建议状态文案：

- `公式未规范化`
- `可阅读，入库前需修复`
- `导出前需校验`

### 入库层

入库前继续做结构化校验：

- 检查 `$...$` 是否成对。
- 检查 `\left` / `\right` 是否配对。
- 检查常见坏 OCR 片段，例如 `\mathfrak{Q}_{\mathrm{R}}B` 这种疑似补集误识别。
- 检查 HTML table、图片引用、公式块是否可被导出管线接受。

校验失败时，不应阻止用户查看题目，但应把题目标为 `需处理`。

### 导出层

导出 LaTeX/PDF 前执行严格校验：

- KaTeX 可渲染不等于 LaTeX 一定可编译。
- 导出前需要使用导出模板实际编译或运行等价校验。
- 失败时定位到题号、字段、公式片段。

导出层不使用宽容降级，因为最终 PDF 不能带坏公式。

## 推荐实施阶段

### 阶段 1：只做诊断增强

不改变现有绑定结果，只增加诊断输出：

- 对每个 OCR image block 记录候选归属题号。
- 记录是否仅因 page span 被归属。
- 对跨页冲突输出 warning。
- 对渲染失败公式记录字段、片段、错误原因。

验收标准：

- 能在第 17/18 题案例中看到第 15 页 image block 同时处于 17/18 交界区。
- 能明确指出当前绑定来自 page span，而不是 bbox 匹配。
- 第 1 题能定位到不平衡公式片段。

### 阶段 2：图块绑定规则切换

将 GLM 图块导入改为：

- 先用 review item `figures_json`。
- GLM 图块必须通过 bbox/segment 匹配才合并。
- 无法匹配的 GLM 图块不进入题目，只进入诊断。

验收标准：

- 第 17 题保留空间图。
- 第 18 题不再出现第 17 题空间图。
- 第 7 题这类本题题干图仍能正常保留。

### 阶段 3：前端宽容渲染

识别结果页和待入库页改为宽容渲染：

- 公式失败时内联降级，不破坏整段。
- 用户能看到模型原文和失败提示。
- 题目状态能反映“可阅读但需格式修复”。

验收标准：

- 第 1 题解析不再大片红字。
- 失败公式片段可读。
- 页面能明确提示入库前需要修复。

### 阶段 4：入库/导出严格校验

建立统一校验结果：

- `previewWarnings`
- `importBlockingIssues`
- `exportBlockingIssues`

验收标准：

- 宽容预览不会自动放行坏公式入库。
- 导出前能定位坏公式所属题号和字段。
- 用户可以在入库确认页集中处理问题题。

## 风险与注意事项

- 过度依赖 OCR image block 会导致错位；过度依赖切题 figures 又可能漏掉 OCR 新发现的图片。因此需要“切题为主，OCR 补充”的策略。
- 前端宽容渲染只能改善阅读体验，不能替代正式 LaTeX 校验。
- 手动复核过的图块优先级应高于自动识别结果。
- 如果用户在切题复核阶段调整过题号或边界，后续 OCR 导入必须使用调整后的 review item，而不是原始 cut result。

## 待确认问题

1. 未匹配的 OCR image block 是否需要在 UI 中提供“手动绑定到第 N 题”的入口。
2. 前端宽容渲染是否默认显示错误提示，还是只在悬停/展开时显示。
3. 入库时遇到公式问题，是阻塞入库，还是允许入库但标记为 `格式需复核`。
4. 导出前是否自动批量尝试修复简单 LaTeX 错误，还是只报告问题。
