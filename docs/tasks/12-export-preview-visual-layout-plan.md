# 导出预览与可视化排版后续执行计划

## 1. 文档目的

本文档用于承接试卷 LaTeX 导出优化的后续开发工作，提供可以逐项执行、验证和打勾的任务清单。

本计划的前置工作已经完成：

- [x] 阶段 1：修正无锚点题干图的默认导出顺序，避免题干图打断 A–D 选项。
- [x] 阶段 2：改进选择题四栏、两栏、单栏自动识别，并统一前端与服务端判断。
- [x] 阶段 3：建立版本化 `PaperLayoutDraft`、`QuestionLayout`、`FigureLayout` 基础模型，并接入可选导出覆盖。

后续工作的核心目标是：让用户在最终导出之前直接看到由 XeLaTeX 生成的 PDF 成品预览，使用受约束的可视化操作调整题目、图片、选项和分页，并让不熟悉 LaTeX 的用户能够像使用 Word 公式一样修改数学内容。

### 1.1 已确认的预览技术决策（2026-07-13）

- [x] 不再使用 HTML/CSS 模拟试卷成品排版。
- [x] PDF 是排版工作台、最终导出和视觉验收的唯一事实来源。
- [x] 工作台显示 PDF 页面或由同一 PDF 渲染出的页面图片。
- [x] 用户操作修改结构化布局草稿，不直接修改 PDF 像素或页面坐标。
- [x] 布局保存后由 XeLaTeX 重新编译，并用新 revision 的 PDF 页面替换旧预览。
- [x] HTML 仍可用于大纲、属性面板、警告列表、加载骨架和公式编辑器，但不承担分页、图文混排或成品版式判断。
- [x] 现有快速预览代码只作为待移除的历史实现，不再扩展功能或作为验收依据。

## 2. 总体原则

- [x] 题库内容与单次试卷排版配置保持分离。
- [x] 排版工作台预览与最终导出均由 XeLaTeX 生成。
- [x] 工作台 PDF 预览与最终 PDF 必须由同一版本的内容快照、布局草稿和模板生成。
- [x] 首版采用受约束排版，不提供任意像素绝对定位。
- [x] 所有自动排版结果都允许单题覆盖，并支持恢复自动值；覆盖结果以重新生成的 PDF 为准。
- [x] 未提供布局草稿时，继续兼容现有一键导出流程。
- [ ] 用户在预览中修改题目内容时，默认只修改当前试卷快照。
- [x] 数据库变更必须采用兼容迁移，不破坏历史集合和导出记录。

## 3. 阶段依赖与交付顺序

建议严格按以下顺序推进：

1. 阶段 4：自动图文混排与排版诊断。
2. 阶段 5：排版草稿持久化与 PDF 预览。
3. 阶段 6：以 PDF 为画布的可视化排版工作台。
4. 阶段 7：富文本与可视化公式编辑。
5. 阶段 8：稳定性、兼容性和正式发布。

开始下一阶段前，应先满足上一阶段的验收清单。

### 当前实现摘要（2026-07-13）

- 阶段 4 已完成布局模型、图片自动决策、左右混排 LaTeX 宏、选项布局统一、XeLaTeX telemetry、页面级二次优化和基础诊断；剩余工作以补齐代表性样例和阶段验收为主。
- 阶段 5 已完成布局草稿表、内容快照、revision 乐观锁、CRUD/预览/导出 API、单 worker 预览队列、重启恢复、旧 revision 清理和只读 PDF 预览页。预览任务会生成 PDF 与页面 PNG，并将状态和诊断保存到明确的草稿 revision。
- 阶段 6 已完成三栏工作台、PDF 页面连续预览、题目顺序调整、选项/分页/答题区/图片属性配置、防抖保存、撤销重做、预览状态轮询、旧 PDF 保留、警告定位高亮和本页/整卷恢复自动。
- HTML A4 快速预览、浏览器分页估算、HTML/PDF 差异比较及其专用测试已经移除；解答题等高排版改为读取 XeLaTeX telemetry。
- 已通过服务端与前端构建、前端组件测试、题型/排版/图片顺序专项测试、布局草稿专项测试、route contract、smoke 和真实 XeLaTeX PDF 端到端回归。包含图片选择题和 16 道解答题的样卷生成学生版 7 页、教师版 3 页，并验证题序、图片槽位、图片宽度、强制分页及最终导出记录绑定同一草稿 revision。

## 4. 阶段 4：自动图文混排与排版诊断

### 4.1 阶段目标

在现有题干图顺序修复和布局模型基础上，实现确定性的图片布局决策。第六题一类“短选项 + 单张方形题干图”应能自动采用左侧选项、右侧图片的稳定版式；其他题型应可靠回退到图片独占一行。

### 4.2 数据模型完善

- [x] 检查并补全 `FigureLayout` 的字段定义。
- [x] 支持 `placement: auto | before-choices | after-choices | side-left | side-right | block`。
- [x] 支持 `widthRatio`，并限制在安全范围，例如 `0.15–1.0`。
- [x] 支持 `alignment: left | center | right`。
- [x] 支持 `keepWithChoices`，控制图片是否与选项整体保持。
- [x] 支持图片布局的自动值、人工覆盖值和最终解析值。
- [x] 对未知图片 ID、无效宽度和不支持的 placement 做安全回退。
- [ ] 为旧布局草稿保留版本兼容与升级函数。

### 4.3 自动布局决策器

- [x] 新增独立的图片布局决策函数，避免将判断散落在 LaTeX 拼接代码中。
- [x] 读取图片实际宽度、高度和宽高比。
- [x] 判断题目是否只有一张可见题干图。
- [x] 判断是否恰好有四个选项。
- [x] 复用阶段 2 的选项视觉宽度估算结果。
- [x] 判断图片是否适合左右混排。
- [x] 为宽图、长图、多图和含表格题设置明确回退规则。
- [x] 返回布局结果及原因，而不只返回 placement 字符串。
- [x] 为自动决策结果增加可诊断字段，例如 `reason` 和 `confidence`。

建议决策结构：

```ts
type FigureLayoutDecision = {
  placement: 'before-choices' | 'after-choices' | 'side-left' | 'side-right' | 'block'
  widthRatio: number
  source: 'auto' | 'manual'
  reason: string
  warnings: string[]
}
```

### 4.4 LaTeX 渲染实现

- [x] 为左右混排新增稳定的模板宏。
- [x] 优先采用固定 `minipage` 或等价结构，不依赖不可控浮动。
- [x] 保证题号、题干、图片和选项的缩进一致。
- [x] 保证左右区域宽度之和及间距不超过 `\linewidth`。
- [x] 图片保持宽高比，不允许拉伸变形。
- [x] 设置图片最小可读宽度和最大高度。
- [x] 选项高度大于图片时允许自然撑开布局。
- [x] 图片高度大于选项时保证下一题不会与图片重叠。
- [x] 教师版答案解析不得进入左右混排区域。
- [x] 无法生成左右混排时自动回退为 `题干 → 图片 → 选项`。

### 4.5 页面空间与二次优化

- [x] 扩展现有 XeLaTeX telemetry，记录题目起止页、页面用量、页面高度和图块高度。
- [x] 定义图片允许缩放的最小宽度。
- [x] 页面空间不足时先尝试安全缩小图片。
- [x] 缩小后仍不足时整体移动图文块，而不是拆开 A–D。
- [x] 禁止普通题干图插入选项块内部。
- [x] 对题目跨页、图片过小、布局回退产生警告。
- [x] 限制自动重排迭代次数，避免编译循环。

### 4.6 排版诊断

- [x] 定义统一的 `LayoutWarning` 类型。
- [x] 支持 `choice-overflow`。
- [x] 支持 `figure-too-small`。
- [x] 支持 `question-split`。
- [x] 支持 `page-overflow`。
- [x] 支持 `missing-figure`。
- [x] 支持 `layout-fallback`。
- [x] 每条诊断包含题目 ID、页码、消息和建议动作。
- [x] 导出失败时保留编译前已发现、可供前端显示的诊断信息。

### 4.7 测试清单

- [x] 第六题固定为回归样例。
- [x] 短选项 + 方形题干图自动采用左右混排。
- [ ] 宽图采用独占一行。
- [ ] 竖长图不会被缩小到不可读。
- [x] 多张题干图不会错误进入单图左右混排。
- [x] 有内联 marker 的题干图保持原位置。
- [x] 无锚点题干图不会打断选项。
- [x] 选项图片保持所属选项语义。
- [x] 教师版解析图保持在解析区。
- [x] 人工 placement 覆盖自动结果。
- [x] 无效人工配置安全回退。
- [x] 服务端 TypeScript 构建通过。
- [x] 专项布局测试通过。
- [x] `git diff --check` 通过。

### 4.8 阶段验收

- [ ] 第六题在默认自动模式下得到可接受版式。
- [ ] A–D 选项在所有测试中保持完整连续。
- [ ] 用户配置可以强制图片居中或左右布局。
- [ ] 自动布局失败时不会导致 LaTeX 编译失败。

## 6. 阶段 6：以 PDF 为画布的可视化排版工作台

### 6.1 阶段目标

在阶段 5 的草稿与 PDF 预览基础上提供受约束的可视化排版。用户可以调整题序、选项栏数、图片槽位、图片大小、分页和答题区，而不需要阅读 LaTeX。工作台不在浏览器中重新实现排版引擎；每次保存后的视觉结果均以当前 revision 重新生成的 PDF 页面为准。

### 6.2 页面结构

- [x] 左侧实现章节与题目大纲。
- [x] 中间显示 PDF 页面或由 PDF 渲染出的页面图片。
- [x] 右侧实现当前元素属性面板。
- [x] 保持现有 AppShell、侧栏和页面头部规范。
- [x] 支持折叠左右面板。
- [x] 支持页面缩放。
- [x] 支持定位当前题目。
- [x] 支持警告定位与高亮。

### 6.3 PDF 预览画布

- [x] 显示当前已保存 revision 对应的 PDF 页面图片。
- [x] 显示学生版和教师版 PDF 预览。
- [x] 支持连续滚动、翻页、缩放和当前页跟踪。
- [x] 编译期间保留上一 revision 的 PDF，并明确显示“预览正在更新”。
- [x] 新 PDF 就绪后只接受与当前草稿 revision 一致的结果。
- [x] 显示 queued、rendering、ready、failed 和过期状态。
- [x] 将 telemetry 中的题目页码映射到左侧大纲和警告列表。
- [x] 点击题目或警告时定位对应 PDF 页，并高亮关联题目状态。
- [x] 移除工作台中的 HTML A4 快速预览入口和实时/精确预览切换。
- [x] 移除浏览器分页估算、HTML/PDF 页数比较及其专用状态。
- [x] PDF 编译失败时保留上一份可用预览，并提供错误说明和重试入口。

PDF 页面本身是只读结果。首版不要求在 PDF 图片上识别任意拖放坐标；题序可在左侧大纲调整，图片槽位、宽度、选项栏数和分页在右侧属性面板调整。每次结构化修改保存后重新生成 PDF。若后续需要直接在 PDF 上操作，只允许基于 telemetry 提供的题目/图片语义区域叠加控件，不引入自由绝对定位。

### 6.4 题目拖拽

- [x] 支持同一章节内调整题目顺序。
- [x] 明确是否允许跨章节拖动。
- [x] 在左侧大纲拖动时显示目标位置。
- [x] 拖动结束后更新 layout draft，而不是题库集合原始顺序。
- [x] 保持题号和分值同步重新计算。
- [x] PDF 页面保持只读，不在页面内容上触发题目排序。
- [x] 支持键盘操作或替代按钮，满足基本可访问性。

### 6.5 图片可视化调整
